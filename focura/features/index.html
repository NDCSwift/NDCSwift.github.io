Focura features, explained factually

Focura is an iOS time-blocking application for adults with ADHD. Every feature below ships in the current App Store version.

A timer that adapts to your struggle

During onboarding, Focura asks whether you struggle most to start, stay focused, or stop. The answer configures the timer's default duration, transition buffer length, and haptic behavior:

Staying25 minutes with anchoring haptics5 minutes
Stopping25 minutes, longer transitions10 minutes

Flow State: hyperfocus is a feature

When a session runs past its planned end, Focura does not sound an alarm or show red warnings. The interface shifts to a green Flow State that counts time past the plan. Sessions ended from Flow State credit the full planned time.

Your calendar becomes the plan

Focura reads iOS Calendar events (read-only) and displays them as locked blocks in the daily timeline, so the day is never empty. If a tracked event starts during an active focus session, Focura automatically pauses the timer and notifies you.

Brain dump via iOS Reminders

Your iOS Reminders appear as an inbox on the Today tab. Tapping one converts it into a time block; completing the session offers to check the reminder off.

Aggressive forgiveness

  • Smart Reschedule (Premium) finds a missed block a new open slot in one tap.
  • Skip reasons record why a block was skipped, so statistics stay honest without shame.
  • Streak freezes (2 per week, free for everyone) prevent one rough day from breaking a streak.
  • Fresh Start clears the rest of the day's plan without erasing completed work.

Always one glance away

A Lock Screen Live Activity, Dynamic Island support, and a Home Screen widget show the current block and daily progress without unlocking the phone. Widgets and Live Activities are free features.

Private by architecture

Focura stores all blocks and sessions in a local SQLite database on the device. There is no account, no cloud sync, and no tracking. The App Store privacy label declares only purchase history (required by the payment processor).
